## 1. 项目概述

构建一个**纯前端交互式 HTML 看板**，完整实现海龟交易策略（Turtle Trading System）的通道计算、ATR 风险管理、信号生成、模拟回测和可视化展示。用户可通过侧边栏调节核心参数（通道周期、ATR 周期、风险上限、止损倍数等），实时观察不同参数下策略收益的变化。

### 1.1 与 task3_dual_ma 的关系

| 维度 | task3_dual_ma（双均线） | task4_turtle（海龟） |
|------|------------------------|----------------------|
| 信号来源 | 均线交叉（金叉/死叉） | 唐奇安通道突破 |
| 仓位管理 | 固定百分比 | ATR 动态仓位（2% 风险 / N） |
| 止损方式 | 固定百分比止损 | 2N 硬止损 + 通道反转退出 |
| 加仓机制 | 无 | 金字塔加仓（每 0.5N 加 1 单位，最多 4 单位） |
| 回测指标 | 10 项 | 12 项（增加盈亏比细分、最大连续亏损次数） |
| 数据源 | 同一批 JSON | 同一批 JSON（复用） |

### 1.2 技术栈

- **生成器**: Python 脚本 (`turtle_dashboard.py`)，将数据嵌入 HTML
- **前端**: 纯 HTML + CSS + JavaScript（无框架依赖）
- **图表库**: Plotly.js 2.35.2（CDN）
- **数据**: 5 只股票前复权日线 JSON + 1 年期/2 年期国债收益率 JSON
- **输出**: 单个 `index.html` 文件，浏览器直接打开即可运行

## 3. 策略引擎

### 3.1 唐奇安通道（Donchian Channel）

```
上轨 = MAX(过去 entry_period 天的最高价)    // 不含当日
下轨 = MIN(过去 entry_period 天的最低价)    // 不含当日
```

**关键规则**：
- 计算上轨/下轨时使用 `high_qfq` / `low_qfq`
- 通道计算窗口为 `entry_period`（默认 20，可选 55）
- **防未来函数**：第 i 天的通道值基于第 `i-entry_period` 到第 `i-1` 天的数据（不含当日）
- 通道在前 `entry_period` 天内为 null（warmup 期）

### 3.2 ATR（Average True Range）

> **关键澄清**：ATR[t] 的计算**包含当日 T 日数据**（high[t]、low[t]），但**不使用当日收盘价 close[t]**。
> TR 公式使用的是 `close[t-1]`（昨日收盘），而非 `close[t]`。
> 因此 ATR[t] 在 t 日收盘前就已完全确定，可在 t 日收盘后用于信号确认和仓位计算，
> 交易在 t+1 日开盘执行——**不存在未来函数问题**。

#### Step 1: 单日 True Range

```
TR[t] = MAX(
    high_qfq[t] - low_qfq[t],              ← 当日振幅（t 日盘中已知）
    |high_qfq[t] - close_qfq[t-1]|,        ← 当日最高 vs 昨日收盘
    |low_qfq[t] - close_qfq[t-1]|          ← 当日最低 vs 昨日收盘
)
```

**注意**：三个分量均使用 `high[t]`、`low[t]`（当日）和 `close[t-1]`（昨日收盘），**不使用 `close[t]`（当日收盘）**。

首日 TR = high - low（无前日收盘可用）。

#### Step 2: ATR 平滑

海龟原版使用 EMA 平滑：

```
ATR[t] = (ATR[t-1] × (period-1) + TR[t]) / period
```

- ATR 周期默认 20（与通道周期一致）
- ATR 初始值 = 前 `period` 天 TR 的简单平均
- ATR 在前 `period` 天内为 null（warmup 期）
- **ATR 每天都要计算**——它是一个滚动指标，不同用途取不同时点的值（见下表）

#### ATR 的不同用途与时点

> **核心区分**：ATR 是每天计算的滚动指标，但不同用途取不同时点的 ATR 值。
> 止损价是"冻结"的历史 ATR，加仓信号检测是"实时"的当日 ATR。

| 用途 | 用哪天的 ATR | 时点说明 |
|------|-------------|----------|
| 入场仓位计算 | 信号日 t 的 ATR[t] | t 日收盘确认信号 → 用 ATR[t] 算仓位 → t+1 开盘执行 |
| **加仓信号检测** | **当天 t 的 ATR[t]** | **必须每天计算**，否则无法判断 close[t] 是否涨够 add_spacing×N |
| 加仓仓位计算 | 加仓信号日 s 的 ATR[s] | s 日收盘确认 → 用 ATR[s] 算仓位 → s+1 开盘执行 |
| 止损价设定 | 入场/加仓信号日的 ATR | 设定后**固定不变**，不随每日 ATR 变化 |
| 止损盘中检测 | **不用 ATR** | 只比较 `low[t]` vs 已固定的止损价 |

**为什么加仓信号用当日 ATR 没有问题**：
- 加仓条件：`close[t] >= 上次买入价 + add_spacing × ATR[t]`
- ATR[t] = f(high[t], low[t], close[t-1]) —— 不使用 close[t]
- close[t] 和 ATR[t] 在收盘时**都已知**，确认后 t+1 开盘执行
- **不存在循环依赖**：ATR[t] 不依赖 close[t]，两者独立可得

### 3.3 交易信号生成

#### 3.3.1 入场信号

| 方向 | 触发条件 | 执行时点 |
|------|----------|----------|
| 做多 | `close_qfq[t] > 上轨[t]`（价格突破 entry_period 日最高价） | t+1 日开盘买入 |
| 做空 | 不支持（A股做空受限） | — |

**信号确认**：第 t 天收盘后确认突破，第 t+1 天开盘执行。这确保不使用当日收盘价做交易决策（防未来函数）。

#### 3.3.2 退出信号（趋势反转通道）

| 退出类型 | 触发条件 | 执行时点 |
|----------|----------|----------|
| 通道退出 | `close_qfq[t] < MIN(过去 exit_period 天的最低价)` | t+1 日开盘卖出 |

- 退出通道周期 `exit_period` 默认 10（对应系统一），可选 20（对应系统二）
- 退出通道同样**不含当日**，基于前 `exit_period` 天

#### 3.3.3 止损信号

> **关键澄清——止损价用的是哪天的 ATR？**
>
> 止损价**不是每天重新计算的**。它在入场时设定一次，之后固定不变，
> 只有在金字塔加仓时才上移。这是海龟原版的"阶梯式止损"，不是每日 trailing stop。
>
> **时间线**：
> - 信号日 t 收盘：确认突破，ATR[t] 已知（用 high[t], low[t], close[t-1] 算出）
> - 执行日 t+1 开盘：买入成交，止损价最终确定 = `open[t+1] - stop_mult × ATR[t]`
> - 之后每一天 t+2, t+3, ...：止损价**保持不变**，仍用 ATR[t]（信号日的 ATR）
> - 加仓信号日 s 收盘：确认加仓，ATR[s] 已知
> - 加仓执行日 s+1 开盘：止损价上移 = `open[s+1] - stop_mult × ATR[s]`，之后再次固定
>
> 因此盘中检测 `low[当前日] <= 止损价` 时，止损价是基于**信号日的 ATR**（一个历史值），
> 不依赖当天的 high/low，不存在循环依赖。

```
止损价 = 入场价(或最近加仓价) - stop_multiplier × ATR[信号确认日]
```

- `stop_multiplier` 默认 2.0（海龟标准 2N 止损）
- 止损价在**入场时设定**（或加仓时更新），之后**固定不变**，不随每日 ATR 重新计算
- 当日 `low_qfq[t] <= 止损价` 时触发，按止损价执行卖出（非开盘价）
- **跳空处理**：如果 `open_qfq[t] < 止损价`（开盘已跳空低于止损），则按 `open_qfq[t]` 执行（无法获得更好的价格）
- 止损触发后当笔交易结束，等待下一次入场信号

> **止损 vs 通道退出的执行时点差异（专业说明）**：
>
> | 信号类型 | 本质 | 确认时点 | 执行时点 | 用的数据 | 原因 |
> |----------|------|----------|----------|----------|------|
> | 止损 | 盘中事件 | 盘中价格触及止损价即触发 | **当日**按止损价执行 | 止损价基于**信号日**的 ATR（历史值，已固定） | 止损单是入场后就挂着的 standing order，盘中触及即成交 |
> | 通道退出 | 收盘事件 | 需要 `close[t]` 才能判断是否跌破通道 | **t+1 日**开盘执行 | 退出条件用 `close[t]`（当日收盘） | 退出条件依赖收盘价，收盘前无法确认 |
>
> 这种不对称是**专业且正确**的：
> - 止损是"已知条件"——止损价在入场时就确定了（基于信号日 ATR），盘中 `low[t]` 触及它即可执行，不需要等收盘，也不依赖当天 ATR
> - 通道退出是"收盘条件"——必须看到 `close[t]` 才知道是否跌破通道下轨，收盘前无法确认
> - **不存在循环依赖**：止损价用历史 ATR（信号日的），盘中检测用 `low[t]`，两者互不影响

#### 3.3.4 信号优先级

同一日内可能同时触发多个信号，按以下优先级处理：

```
止损 > 通道退出 > 入场信号
```

即：如果当日同时触发止损和通道退出，执行止损；如果持仓中触发止损，不再执行入场信号。

### 3.4 仓位管理

#### 3.4.1 初始仓位计算

```
单笔风险金额 = 账户总资金 × risk_per_trade / 100
单位仓位股数 = 单笔风险金额 / (stop_multiplier × ATR[信号确认日t])
```

- `risk_per_trade` 默认 2%（海龟标准）
- ATR[t] 是信号确认日（t 日收盘）的 ATR，此时已完全确定
- 股数向下取整到 100 股（A股最小交易单位 1 手）
- 如果计算股数 < 100，则不入场（风险无法控制到 2% 以内）

#### 3.4.2 金字塔加仓

```
每次加仓条件: close[s] >= 上次买入价 + add_spacing × ATR[s]    // s = 加仓信号日
加仓股数: 单笔风险金额 / (stop_multiplier × ATR[s])            // 用加仓信号日的 ATR
最大持仓单位: 4 个单位
```

加仓时：
- 用加仓信号日 s 的 ATR[s] 重新计算加仓股数（ATR 可能已变化）
- **上移止损价** = open[s+1] - stop_multiplier × ATR[s]（止损随加仓上移，之后再次固定）
- 加仓后总仓位不超过 4 个单位

#### 3.4.3 仓位上限

- 单只股票最大投入 = 账户总资金 × max_position_pct / 100
- `max_position_pct` 默认 80%
- 如果加仓后超过此上限，则不执行该次加仓

## 4. 回测引擎

### 4.1 回测流程

```
// 初始化：止损价 = null（空仓时无止损）

for each day t in backtest_period:
    1. 计算当日通道上轨/下轨（基于 t-period 到 t-1 的数据，不含 t 日）
    2. 计算当日 ATR[t]（用 high[t], low[t], close[t-1]，盘中即可确定）

    3. 执行前一日标记的交易（t 日开盘执行）：
       if 标记了买入：
           买入价 = open_qfq[t] × (1 + 佣金 + 滑点)
           仓位 = risk_amount / (stop_mult × ATR[昨日信号日])    ← 信号日的 ATR，非今日
           止损价 = 买入价(原始) - stop_mult × ATR[昨日信号日]   ← 设定后固定
       if 标记了加仓：
           加仓价 = open_qfq[t] × (1 + 佣金 + 滑点)
           加仓仓位 = risk_amount / (stop_mult × ATR[昨日信号日])
           止损价 = 加仓价(原始) - stop_mult × ATR[昨日信号日]   ← 上移后固定

    4. 检查止损（盘中事件，当日执行）：
       if 持仓 and 止损价 != null and low_qfq[t] <= 止损价:
           执行价 = max(止损价, open_qfq[t])  // 跳空时取开盘价
           按执行价平仓，记录交易（退出类型="止损"）
           止损价 = null

    5. 检查退出信号（收盘事件，次日执行）：
       if 持仓 and close_qfq[t] < 退出通道下轨[t]:
           标记 t+1 日开盘卖出（退出类型="通道退出"）

    6. 检查入场信号（收盘事件，次日执行）：
       if 空仓 and close_qfq[t] > 入场通道上轨[t]:
           标记 t+1 日开盘买入（记录信号日 = t，供明日执行时用 ATR[t]）

    7. 检查加仓信号（收盘事件，次日执行）：
       if 持仓 and 加仓次数 < 4 and close_qfq[t] >= 上次买入价 + add_spacing × ATR[t]:
           标记 t+1 日开盘加仓（记录信号日 = t，供明日执行时用 ATR[t]）

    8. 计算当日净值 = cash + shares × close_qfq[t]
```

> **ATR 使用时点总结**：
> - 仓位计算和止损价设定：用**信号确认日**的 ATR（t 日收盘确认 → ATR[t] → t+1 开盘执行时使用）
> - 止损盘中检测：用**入场时已固定**的止损价（基于信号日 ATR，不随当日变化）
> - 通道计算：用 t-period 到 t-1 的数据（不含 t 日）

### 4.2 交易成本模型

与 task3_dual_ma 保持一致：

| 成本项 | 默认值 | 说明 |
|--------|--------|------|
| 佣金 | 0.03% | 买卖双边 |
| 滑点 | 0.01% | 买卖双边 |
| 印花税 | 0.05% | 仅卖出方缴纳 |

> **印花税政策确认**：根据财政部、国家税务总局 2023 年 8 月 28 日起实施的最新政策，
> A 股证券交易印花税由 0.1%（千分之一）减半至 **0.05%（万分之五）**，仅向卖出方征收。
> 买入方不缴纳印花税。本看板按此最新政策执行。

```
买入实际价格 = 开盘价 × (1 + 佣金 + 滑点)
卖出实际价格 = 开盘价 × (1 - 佣金 - 滑点 - 印花税)
```

### 4.3 回测周期

- **默认回测区间**: 近 2 年（约 504 个交易日），覆盖足够多的趋势和震荡周期
- 可通过侧边栏下拉框选择（近 1 年 252 / 近 2 年 504 / 全部）
- **回测区间与无风险利率匹配**：
  - 选"近 1 年" → 夏普比率使用 1 年期国债收益率（回测起始日的 1Y 国债）
  - 选"近 2 年" → 夏普比率使用 2 年期国债收益率（回测起始日的 2Y 国债）
  - 选"全部" → 夏普比率使用 2 年期国债收益率（数据起始日的 2Y 国债）
- 回测起始日之前的通道和 ATR 仍需正常计算（warmup 期在回测开始前完成）

### 4.4 买入持有基准

- 在回测起始日以开盘价满仓买入
- 使用与策略相同的仓位比例（max_position_pct）
- 持有至回测结束日
- 净值曲线作为策略对比基准

## 5. 指标体系

### 5.1 核心指标（12 项）

| # | 指标名 | 计算方式 | 显示格式 |
|---|--------|----------|----------|
| 1 | 总收益率 | (期末净值/期初 - 1) × 100% | +XX.XX% |
| 2 | 年化收益率 | (期末/期初)^(252/天数) - 1 × 100% | +XX.XX% |
| 3 | 最大回撤 | max[(峰值-谷值)/峰值] × 100% | XX.XX% |
| 4 | 胜率 | 盈利交易笔数 / 总交易笔数 × 100% | XX.X% |
| 5 | 盈亏比 | 平均盈利金额 / 平均亏损金额 | X.XX |
| 6 | 夏普比率 | (年化收益 - 无风险利率) / 年化波动率 | X.XX |
| 7 | 超额收益 vs 买入持有 | 策略总收益 - 买入持有总收益 | +XX.XX% |
| 8 | 买入持有收益 | (期末买持净值/期初 - 1) × 100% | +XX.XX% |
| 9 | 年化波动率 | 日收益率标准差 × √252 × 100% | XX.X% |
| 10 | 交易次数 | 总交易笔数（一买一卖为一笔） | XX 笔 |
| 11 | 平均持仓天数 | 所有交易持仓天数均值 | XX 天 |
| 12 | 最大连续亏损次数 | 连续亏损交易的最大连续次数 | XX 次 |

### 5.2 无风险利率

根据回测区间长度匹配对应期限的国债收益率：

| 回测区间 | 国债期限 | 数据文件 | 说明 |
|----------|----------|----------|------|
| 近 1 年（252 交易日） | 1 年期 | `bond_yield_1y.json` | 取回测起始日（约 1 年前）的 1 年期国债收益率 |

- 震荡盘整市（通道反复突破-回撤，频繁止损）
- 低波动蓝筹（趋势空间不足以覆盖成本）
- 短线交易（通道滞后性大）

- 系统一（20日通道）vs 系统二（55日通道）的权衡
- ATR 周期与通道周期的配合
- 止损倍数的选择：太近易被噪音震出，太远风险过大
- 加仓间距的影响：0.5N 是海龟标准，加大间距减少加仓次数

**五、风险管理要点**
- 2% 风险上限是铁律
- 金字塔加仓的前提是趋势确认
- 连续止损时的心理承受
- 仓位上限防止过度集中在单一标的

**六、从本看板回测观察到的规律**
- 不同股票的通道策略表现差异
- 通道周期对交易频率和胜率的影响
- ATR 仓位管理在不同波动率标的上的效果
- 策略 vs 买入持有在不同市场环境下的优劣

## 10. 防未来函数检查清单

| # | 检查项 | 规则 |
|---|--------|------|
| 1 | 通道计算 | 第 t 天通道基于 t-period 到 t-1 的数据，**不含 t 日** |
| 2 | ATR 计算 | **每天计算**。TR[t] 使用 high[t]、low[t]（当日）+ close[t-1]（昨日收盘），**不使用 close[t]**（当日收盘）。ATR[t] 在 t 日盘中即可确定 |
| 3 | 入场信号 | t 日收盘确认突破（close[t] > 上轨[t]），t+1 日开盘执行 |
| 4 | 通道退出信号 | t 日收盘确认反转（close[t] < 下轨[t]），t+1 日开盘执行 |
| 5 | 止损执行 | t 日盘中 low[t] 触及止损价即触发，**当日**按止损价执行。止损价基于**信号确认日**的 ATR（入场/加仓时设定，之后固定不变），**不依赖当日 ATR**，不存在循环依赖 |
| 6 | 加仓信号 | t 日收盘确认涨 add_spacing×N（close[t] >= 上次买入价 + add_spacing×**ATR[t]**）。**用当天 ATR**，必须每天计算。ATR[t] 不含 close[t]，与 close[t] 无循环依赖，t+1 开盘执行 |
| 7 | 仓位计算 | 入场/加仓仓位基于**信号确认日**的 ATR（t 日收盘确认 → ATR[t] → t+1 日开盘执行时使用） |
| 8 | 净值计算 | 使用当日收盘价估值，不含未来信息 |

> **ATR 的"冻结"与"实时"双轨制**：
> - **止损价**：用信号确认日的 ATR，设定后**冻结**（阶梯式，只在加仓时跳升）
> - **加仓信号检测**：用**当天**的 ATR[t]，**每天实时计算**（判断 close[t] 是否涨够 0.5N）
> - **不存在循环依赖**：
>   - 止损检测：low[t] vs 冻结的历史止损价 → 互不影响
>   - 加仓检测：close[t] + ATR[t]，但 ATR[t] 不含 close[t] → 两者独立可得，收盘时都已知

## 11. 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 通道 warmup 期（前 N 天） | 不生成信号，净值 = 初始资金 |
| ATR = 0（停牌后复牌） | 跳过该日信号，仓位计算沿用上一个有效 ATR |
| 计算仓位 < 100 股 | 不入场，记录到日志（"资金不足以开仓"） |
| 一字涨停无法买入 | 检查 `open_qfq == close_qfq == high_qfq`，跳过买入 |
| 停牌日 | `vol == 0` 或 `high == low == close`，净值沿用前日，不执行交易 |
| 回测结束时仍有持仓 | 按最后一日收盘价平仓，记录为"期末平仓" |
| 加仓后总仓位超过 max_position_pct | 不执行该次加仓 |
| 止损跳空（open < 止损价） | 按 `open_qfq[t]` 执行（开盘已低于止损价，无法获得止损价成交） |
